# Roadmap — MedAgenda

## Versão 1 — MVP Funcional

O objetivo desta versão é entregar a funcionalidade essencial: uma agenda funcional, usável e confiável.

**Escopo:**
- Autenticação de usuário (login e cadastro via Supabase)
- Criação, edição e exclusão de compromissos
- Visualização por semana e por mês
- Categorização dos compromissos (Aula, Plantão, Ambulatório, etc.)
- Definição de data, hora de início e hora de término
- Persistência dos dados no Supabase
- Deploy via GitHub Pages

---

## Versão 2 — PWA e Melhorias de UX

O objetivo desta versão é tornar o MedAgenda uma experiência mais fluida e acessível, especialmente em dispositivos móveis.

**Escopo:**
- Transformação em Progressive Web App (PWA)
- Instalação como aplicativo na tela inicial do celular
- Suporte a uso offline básico (visualização de compromissos já carregados)
- Melhorias de UX com base no uso real da Versão 1
- Notificações push para lembretes de compromissos
- Recorrência de compromissos (semanal, mensal)

---

## Versão 3 — Compartilhamento e Integrações

O objetivo desta versão é expandir o MedAgenda para além do uso individual.

**Escopo:**
- Compartilhamento de agenda entre usuários (ex: turmas, grupos de estudo)
- Exportação da agenda para calendários externos (Google Calendar, iCal)
- Sincronização avançada entre dispositivos
- Integrações futuras com plataformas de ensino médico
- Exploração de funcionalidades colaborativas

---

> O roadmap é um guia de direção, não um contrato. As prioridades podem evoluir conforme o feedback dos usuários e as descobertas durante o desenvolvimento.
